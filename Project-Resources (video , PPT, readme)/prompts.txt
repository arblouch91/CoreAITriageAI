    }
  ]
}

Score RangeLabel0.00 – 0.25Extremely Negative0.26 – 0.50Negative0.51 – 0.75Neutral0.76 – 1.00Positive

Digital Banking Support



Relationship Management



Card Operations

The LLM returns a chat completion object. Extract the JSON string from:

response.choices[0].message.content

Parse that string as JSON. It will contain: name, email, complaint, department, priority, sentiments_score, sentiments_label.

## Anti-Patterns ❌

**Avoid:**
- Default Bootstrap blue anywhere — override all primary colors with navy/gold
- Hardcoded URLs — always use `asp-controller`, `asp-action`, `asp-for` Tag Helpers
- Arbitrary spacing (not multiples of 4px or 8px)
- More than one accent color — gold only for CTAs and highlights
- Icons without semantic meaning — don't sprinkle icons randomly
- Unstyled form inputs — all use `.sw-input`, `.sw-select`, `.sw-textarea`
- Tables without responsive wrapper — always `overflow-x: auto` on mobile
- Mobile-last design — design responsive from the start
- `@Html.BeginForm()` — use `<form asp-*>` Tag Helpers only
- Inline styles — all styles in `wwwroot/css/`
- jQuery or heavy JS frameworks — vanilla ES6+ only

## Ambiguity Handling

When requirements are vague, make **reasonable assumptions upfront** rather than over-questioning:

- **Color not specified?** Use navy for backgrounds, gold for CTAs
- **Layout not specified?** Use card-based layout with 24px padding
- **Spacing not specified?** Use 16px–24px gaps between major sections
- **Mobile not mentioned?** Design responsive (mobile < 640px, desktop ≥ 1024px)
- **Validation not mentioned?** Include field-level validation with `.sw-invalid-feedback`
- **Icons not mentioned?** Use Lucide icons for nav items, buttons, and section headings

### 6. PROJECT PURPOSE & MAIN FEATURES

**Problem Statement (Hackathon Challenge C-04):**
Banks receive thousands of customer complaints across multiple channels, creating backlogs and delaying resolution. Manual triage damages customer satisfaction and regulatory standing.

**Solution:** AI-powered complaint triage system that:
- Automatically classifies complaints into predefined categories
- Assigns priority levels (High/Medium/Low)
- Routes to appropriate departments
- Performs sentiment analysis (0.00–1.00 scale)
- Auto-drafts professional, empathetic responses
- Provides a dashboard for staff to manage the complaint queue

**Core Features:**

| Feature | Description |
|---------|-------------|
| **Complaint Submission** | Customers submit complaints via form (Name, Email, Phone, Complaint Text) |
| **AI Analysis** | Single LLM call performs: category classification, department routing, priority assignment, sentiment analysis, and draft response generation |
| **Dashboard** | Staff view all complaints with filters, sorting, pagination, and aggregate statistics |
| **Filtering & Sorting** | By Department, Priority, Category, Status; sort by Date, Priority, Category, Status |
| **Pagination** | 10 complaints per page with navigation |
| **Statistics** | Total complaints, high-priority count, average sentiment score |
| **Status Tracking** | Open, In Progress, Resolved |

**Available Categories:** Fraud, Fees, Service Downtime, Card Issue

**Available Departments:** Fraud / Billing Team, Digital Banking Support, Relationship Management, Card Operations

**Available Statuses:** Open, In Progress, Resolved

- **Purpose:** Integrates with OpenRouter LLM API for complaint analysis
- **Key Method:** `AnalyzeAsync(string name, string email, string complaint)` → `Task<ComplaintAnalysis>`
- **Workflow:**
  1. Builds a detailed prompt with complaint details and available options
  2. Makes POST request to `https://openrouter.ai/api/v1/chat/completions` with Bearer token authentication
  3. Extracts JSON from response (strips markdown fences if present)
  4. Deserializes to `ComplaintAnalysis` object
- **Response Structure:**
  ```json

    "department": "Fraud / Billing Team|Digital Banking Support|Relationship Management|Card Operations",
  ```
